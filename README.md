# BDD Agent

An AI-powered agent that generates, reviews, and runs BDD test cases from an API specification.

## How It Works

```
API Spec Input
     ↓
[Generate Tests]  → LLM generates Gherkin scenarios
     ↓
[Human Review]    → You approve/reject each scenario  ← INTERRUPT
     ↓
[Generate Steps]  → LLM writes Behave step definitions
     ↓
[Run Tests]       → Behave executes the .feature file
     ↓
[Report]          → Comprehensive results displayed + saved
```

## Setup

### 1. Install dependencies

```bash
cd bdd_agent
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY (or ANTHROPIC_API_KEY)
```

### 3. Run the agent

```bash
# Interactive mode (uses built-in example spec)
python main.py

# Load spec from file
python main.py --spec path/to/your/api_spec.txt

# Pass spec inline
python main.py --spec-inline "REST API: My API, Base URL: http://api.example.com ..."
```

## Project Structure

```
bdd_agent/
├── agent/
│   ├── graph.py              # LangGraph state graph
│   ├── llm.py                # LLM factory (OpenAI / Anthropic)
│   ├── state.py              # Shared AgentState TypedDict
│   └── nodes/
│       ├── generate_tests.py # Node 1: LLM → Gherkin scenarios
│       ├── human_review.py   # Node 2: Human approval (interrupt)
│       ├── generate_steps.py # Node 3: LLM → step definitions
│       ├── run_tests.py      # Node 4: Execute Behave
│       └── report.py         # Node 5: Format & save results
├── features/
│   ├── generated.feature     # Auto-generated by agent
│   ├── environment.py        # Auto-generated Behave hooks
│   └── steps/
│       └── generated_steps.py  # Auto-generated step defs
├── prompts/
│   ├── gherkin_prompt.txt    # Prompt for scenario generation
│   └── steps_prompt.txt      # Prompt for step def generation
├── reports/
│   ├── behave_results.json   # Raw Behave JSON output
│   └── summary.txt           # Human-readable report
├── main.py                   # Entry point
├── requirements.txt
└── .env.example
```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `OPENAI_API_KEY` | — | Required if using OpenAI |
| `ANTHROPIC_API_KEY` | — | Required if using Anthropic |
| `LLM_PROVIDER` | `openai` | `openai` or `anthropic` |
| `LLM_MODEL` | `gpt-4o` | Model name |

## Human Review

When the agent pauses for review, you'll see each scenario displayed with its full Gherkin text.
For each scenario you can:
- **Approve** — it will be included in the test run
- **Reject** — optionally add feedback (currently logged, future: fed back to LLM for regeneration)

## Output Files

After a run:
- `features/generated.feature` — the Gherkin feature file
- `features/environment.py` — Behave hooks
- `features/steps/generated_steps.py` — Python step definitions
- `reports/behave_results.json` — raw Behave JSON
- `reports/summary.txt` — formatted report
